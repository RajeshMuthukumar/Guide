Overview

Softpay is a revolutionary mobile application that allows a merchant to turn a commercial off-the-shelf mobile phone into a contactless payment terminal, without requiring additional dedicated hardware. For merchants, this offers mobile payment terminals at a different price-point than existing mobile hardware terminals for EMV transactions.

How it works

The Softpay app can work independently to accept transactions. If you already have a POS-setup, we can easily integrate, customize, and tailored with your application and allows you to start receiving payments by simple App-switch. You can contact us for further information or sign up on https://developer.softpay.io/ Softpay app connects via the Internet and allows Salesperson to trigger Contactless transactions via NFC. Softpay app can trigger Purchase, Refund, Reversal. Softpay app supports Dankort, Visa Dankort, Visa, Master card and Maestro. Talk with your Acquirer for making further agreements.

Which devices does the mobile app support?

You can install Softpay App on any mobile devices that have Android (mobile phones and tablets) OS version > 8.0 and the Device should have below setup.

  • For better working of the app, NFC/Android Bean should be Disabled.
  • Internet either Via WIFI or Mobile network should be connected.
  • NFC should be enabled.
    • Developer Option should be disabled.

Getting Started

Device setup

  1. NFC should be turned ON. Setting ➔ Device connections ➔ NFC [setting changes between different device models.]
  2. Internet should be connected via WIFI or Mobile network

The mobile status should show.

Installation

To find and install Softpay app for Android:

  1. On your Android phone, open Google play Store [or Via a trusted source].
  2. Tap the Search icon.
  3. Enter Softpay in the search field.
  4. Select Softpay in the search results to go to the app page.
  5. Follow the standard installation procedure.

Login and Select Store

To use the app, you need to sign in using the email address and password of your Merchant account. This Login would be a one-time process.

  1. Open the downloaded Softpay app.
  2. Click “Next” on the welcome screens.
  3. Enter the User id and Password. This credential can be collected from Softpay Team.
  4. The operator selects a store from the list of stores shown on the app.
  5. The app initializes with the merchant detail.
  6. Enter amount screen is displayed.

Now the App is ready for receiving payments.

Guidelines

Merchant Guidelines

  • Ensure that only designated employees (have physical and/or) logical access to the payment functionality of the solution.
  • Merchants are encouraged to lock their consumer mobile devices when not in use. The consumer mobile device should be configured to auto-lock after a certain number of minutes of inactivity.
  • Merchants should not circumvent any security measures on the device.
  • Merchants should regularly update the firmware and install any application updates when available.
  • Protect the device from malware and other threats.
  • Do not Unlock bootloader or root the device.
  • Immediately report the loss or theft of a consumer mobile device.
  • If the device leaves the hands of the merchant, the Softpay application should be wiped (uninstall or clear data) or it should be blocked in the Softpay TMS system

The security goals for merchants are as follows:

  • Ensure the secure use of mobile payment acceptance solutions.
  • Limit the exposure of account data that may be used to commit fraud.
  • Prevent software attacks on consumer mobile devices.

Salesperson Guidelines

  • The salesperson shall instruct the consumer to cover their PIN entry, i.e., by keeping it close to their body and/or cover by hand.
  • The salesperson shall not look at the device when the consumer is entering the PIN.
  • The salesperson shall scan the surroundings for other people observing the entry.

Softpay Features

Purchase

  • If the screen is not in enter amount screen Click on the Side menu and click on Payment.
  • On the Enter amount screen, salesperson enters the transaction amount using the Amount keypad.
  • Click on check mark to continue/proceed with the transaction.
  • Customer/Card holder can now “Place the card on the back of the phone”. The timer for tapping the card is 60 seconds, after that the transaction times out.
  • On successful card read, Transaction is forwarded for further processing.
  • Success/failure of transaction reaches Softpay app. Transaction details shows card used, Date, time, Transaction amount and Status of the card.

Refund

Used when goods that have been paid for by credit card are returned. Product refunds are not available for purchases made with debit cards (bank cards).

Reversal

  1. Click on the Side menu and click on Refund. Clear indication of Refund is shown on the Enter amount screen.
  2. On the Enter amount screen, salesperson enters the Refund amount using the Amount keypad.
  3. Click on check mark to continue/proceed with the transaction.
  4. App would request for Biometric authentication from Salesperson/ Superior in the store.
  5. On providing the authentication, Customer/Card holder can now “Place the card on the back of the phone”. The timer for tapping the card is 60 seconds, after that the transaction times out.
  6. On successful card read, Transaction is forwarded for further processing.
  7. Success/failure of transaction reaches Softpay app. Transaction details shows card used, Date, time, Transaction amount and Status of the card.

Reconcialiation

Manual Reconcialiation

Reconciliation transfers transactions to the settlement account on the following business day. The terminal should be reconciled every day. If seven days have elapsed since the last reconciliation, the terminal must be reconciled before it can be used.

Automatic Reconcialiation

Automatic reconciliation involves the terminal carrying out re conciliation at a fixed time every day. If the merchant has several terminals, automatic reconciliation will take place from the same terminal every time. Please contact Nets customer service to activate automatic reconciliation.

Copy of Last Reconcialiation

The terminal saves only the last reconciliation with accumulated transactions. Copy of 0,- reconciliation will not be stored. The same functionality is also implemented for copy of Z-total. This is only implemented for stand-alone terminals. This function is used to extract a copy of the most recent reconciliation report.

Reports

Transaction List

Used to print out a list of all transactions stored in the terminal.

Last Transaction

Used to print out Last transactions stored in the terminal.

New Software

You are able to check if there is any new version of Softpay app available on play store.

  • Enabled – if there is new Softpay update is available the option would be Enabled. On clicking the option, the app routes to Google play store
  • Disabled – If there is no Softpay update is available the option would be disabled.

Transaction Outcome

Status Explanation
Approved Transaction [Purchase/Refund] approved
Declined Transaction [Purchase/Refund] declined
Technical failure Technical failure to process the transaction, try again after some time
Purchase interrupted Purchase interrupted due to network, try again after some time
Pin exceeded Card Pin blocked due to exceeded wrong pin
Incorrect pin Transaction declined due to wrong pin
Insufficient funds Insufficient balance/credit limit reached
Max limit reached Transaction rejected due to Contactless limit reached
Card unknown Unknown card, Try again with some other card
Invalid card Invalid card, Try again with some other card
Expired card Card expired, try again with some other card
See phone See phone for further processing [or] Enter passcode on mobile & tap again
Softpay Not ready Softpay app not ready. Try to force stop the app and try again
No internet Re connect to WIFI/Mobile data and try again
NO NFC Enable NFC and try again
Timed out Issuer not available/Transaction not completed in time
